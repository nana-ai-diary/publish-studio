# Publish Studio

一个面向公众号、小红书图文和长文发稿的本地 Markdown 排版工具。

它参考了 `fawen.fun` 和 `md.doocs.org` 这类轻量编辑器的使用体验：先写内容，再把文章排成图片，或复制为适合公众号编辑器粘贴的富文本。

## 在线使用

部署到 GitHub Pages 后可直接访问：

```text
https://nana-ai-diary.github.io/publish-studio/
```

也可以本地打开 `index.html`，或启动一个本地服务：

```powershell
py -3 -m http.server 5173
```

然后访问：

```text
http://127.0.0.1:5173/
```

## 功能

- 本地文档库：新建、复制、删除、切换历史文档
- 自动保存：正文、作者信息、头像、配图和样式保存到浏览器 `localStorage`
- 默认使用指南：首次打开即展示“娜娜酱AI日记”示例文档
- Markdown 排版：H1、H2、加粗、斜体、引用、代码块、居中标题块
- 颜色工具：正文色、标题色、选中文字颜色、选中文字高亮
- 查找替换：查找、上一个、下一个、替换当前、全文替换
- 导入导出：导入 JSON 备份、Markdown 文档；导出 JSON 备份、Markdown、Word
- 对齐工具：左对齐、居中、右对齐，可替换已有对齐标记
- 强制分页：插入分页标记，把后续内容推到下一张图
- 图片素材管理：上传、插入、删除、清空、裁剪、恢复原图
- 多图排版：单图、双图左右、双图上下、三图左1右2、三图右1左2、三图并列、四图环绕
- 裁剪比例：原比例、1:1、3:4、4:3、16:9
- 图片导出：单张 PNG、长图、批量 ZIP
- 富文本复制：写入 `text/html` 和 `text/plain`，适合粘贴到微信公众号编辑器

## 常用语法

### 标题和正文

```md
# 一级标题
## 二级标题

这是普通正文，支持 **加粗** 和 *斜体*。
```

### 高亮和文字颜色

```md
==这是一段高亮文字==
{color:#e11d48|这是一段自定义颜色文字}
```

### 引用

```md
> 这是一段引用。
```

### 代码块

````md
```js
const title = "娜娜酱AI日记";
console.log(title);
```
````

### 居中标题块

```md
:::title
欢迎来到娜娜酱AI日记
:::
```

### 对齐段落

```md
:::align center
这一行会居中显示
:::
```

可选值：`left`、`center`、`right`。

### 强制分页

当某一段内容需要从下一张图开始时，可以插入：

```md
:::pagebreak
```

工具栏里的分页按钮会自动插入这个标记。

### 导入导出

左侧的“导出”支持三种格式：

- `JSON 备份`：导出所有本地文档、图片素材和样式设置，适合迁移和恢复。
- `Markdown`：导出当前文档的 `.md` 文件，适合继续编辑或同步到其他 Markdown 工具。
- `Word`：导出当前文档的 `.doc` 文件，适合用 Microsoft Word 或 WPS 打开。

左侧的“导入”支持：

- `publish-studio` 导出的 JSON 备份
- 普通 `.md` / `.markdown` / `.txt` 文档

导入采用追加模式，不会清空当前已有文档；可以一次选择多个 Markdown 文档导入。

### 图片标记

上传图片会自动插入图片标记：

```md
[[image:img_id]]
```

多图会自动插入图库标记：

```md
[[gallery:img_a,img_b,img_c|3|three-right]]
```

常见版式：

- `two-cols`：双图左右，默认 1:1
- `two-rows`：双图上下，默认 16:9
- `three-left`：三图左1右2，小图默认 1:1
- `three-right`：三图右1左2，小图默认 1:1
- `three-cols`：三图并列，默认 1:1
- `four-grid`：四图环绕，默认 1:1

## 数据说明

所有文档默认保存在当前浏览器的 `localStorage` 中，不会上传到服务器。

如果清空浏览器缓存或换浏览器，历史文档会丢失。后续可以扩展“导入/导出备份”功能。
